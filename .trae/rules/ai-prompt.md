# AI 提示词工程仓库项目约束

## 1. 项目简介

本仓库用于存储和管理 AI 相关的提示词工程（Prompt Engineering）资源，包括各类场景的提示词模板、最佳实践。

## 2. 目录结构规范

```
SmileX-AI-Prompt/
├── .trae/
│   └── rules/
│       └── ai-prompt.md          # 项目约束文件
└── README.md                     # 项目说明
```

> 提示：随着项目发展，可按需扩展以下目录结构：
> - `prompts/` - 提示词主目录（chat、generation、analysis、coding、translation）
> - `templates/` - 提示词模板

## 3. 提示词编写规范

### 3.1 文件命名规范
- 使用小写字母和下划线命名
- 包含清晰的功能描述
- 示例：`chatgpt_code_review.md`、`midjourney_landscape.md`

### 3.2 提示词结构
每个提示词文件应包含以下元信息：

```markdown
---
name: 提示词名称
version: 1.0.0
author: 作者名
date: 2024-01-01
tags: [标签1, 标签2]
description: 简短描述
model: 适用模型
---
```

### 3.3 提示词内容要求
- 清晰定义角色和任务
- 提供具体的输入示例
- 说明输出格式要求
- 包含必要的约束条件
- 必要时添加思维链引导

## 4. 版本管理规范

### 4.1 版本号格式
采用语义化版本号：`主版本.次版本.修订号`

- 主版本：不兼容的重大变更
- 次版本：向后兼容的新功能
- 修订号：向后兼容的问题修复

### 4.2 更新记录
每次更新提示词时，在文件头部添加更新日志：

```markdown
## 更新日志
- 2024-01-01: v1.0.0 - 初始版本
- 2024-01-15: v1.1.0 - 添加xxx功能
```

## 5. 性能与安全

### 5.1 提示词优化
- 避免冗余和重复的指令
- 使用清晰、具体的语言
- 考虑令牌（Token）使用效率

### 5.2 敏感信息处理
- 不在提示词中包含敏感信息
- 使用占位符替代实际密钥
- 遵循数据安全最佳实践
