# Submodule Updater

## 功能描述

本技能用于自动更新 Git 子模块（submodule），支持两种模式：
1. **更新所有子模块**：更新项目中的所有子模块
2. **更新指定目录下的子模块**：仅更新指定路径下的子模块

## 触发条件

当用户发出以下指令时，必须立即调用此技能：
- "更新所有submodule"
- "更新[指定目录]下的submodule"
- "update all submodules"
- "update submodules in [directory]"
- 或其他类似的子模块更新请求

## 指令解析规则

### 模式1：更新所有子模块
识别以下指令模式：
- "更新所有submodule"
- "更新全部submodule"
- "update all submodules"
- "update every submodule"

**执行命令**：
```bash
git submodule update --remote --merge
```

### 模式2：更新指定目录下的子模块
识别以下指令模式：
- "更新[目录路径]下的submodule"
- "update submodules in [directory]"
- "update [directory] submodules"

**目录路径提取规则**：
1. 从指令中提取方括号内的路径
2. 路径可以是相对路径或绝对路径
3. 支持正斜杠（/）和反斜杠（\）

**执行命令**：
```bash
git submodule update --remote --merge [目录路径]
```

## 执行流程

### 步骤1：指令解析
- 分析用户输入的指令
- 识别是更新全部还是指定目录
- 提取目录路径参数（如果存在）

### 步骤2：验证环境
- 检查当前目录是否为 Git 仓库
- 验证是否存在 .gitmodules 文件
- 验证指定目录是否存在（如果是指定目录模式）

### 步骤3：执行更新
- 执行 `git submodule update --remote --merge` 命令
- 捕获命令输出和错误信息

### 步骤4：实时反馈
- 显示正在更新的子模块名称
- 显示更新进度
- 显示每个子模块的更新结果

### 步骤5：状态报告
- 统计成功更新的子模块数量
- 统计失败的子模块数量
- 显示详细的错误信息（如果有）
- 提供操作完成的总耗时

## 输出格式

### 成功情况
```
✓ 开始更新子模块...
✓ 正在更新: [子模块名称]
✓ 子模块 [子模块名称] 更新成功
...
✓ 所有子模块更新完成
✓ 成功: [数量] 个
✗ 失败: [数量] 个
⏱ 总耗时: [时间]
```

### 失败情况
```
✗ 错误: [错误信息]
✓ 成功: [数量] 个
✗ 失败: [数量] 个
⏱ 总耗时: [时间]
```

## 错误处理

### 常见错误及处理
1. **不是 Git 仓库**
   - 错误信息："当前目录不是 Git 仓库"
   - 处理：终止操作并提示用户

2. **不存在子模块**
   - 错误信息："未找到任何子模块"
   - 处理：提示用户检查 .gitmodules 文件

3. **目录不存在**
   - 错误信息："指定的目录不存在：[路径]"
   - 处理：提示用户检查路径是否正确

4. **网络错误**
   - 错误信息："网络连接失败"
   - 处理：提示用户检查网络连接

5. **权限错误**
   - 错误信息："权限不足"
   - 处理：提示用户检查文件权限

## 实现要求

1. **指令识别**：必须准确识别用户的更新指令
2. **路径提取**：正确提取用户指定的目录路径
3. **实时反馈**：提供操作过程的实时输出
4. **错误捕获**：捕获并显示所有可能的错误
5. **状态统计**：统计并显示成功和失败的数量
6. **性能优化**：使用并发更新（如果可能）以提高效率

## 注意事项

1. 优先使用 HTTPS 协议执行 git submodule add 命令
2. 在执行更新前，建议先备份当前状态
3. 如果子模块有未提交的更改，可能需要先提交或暂存
4. 更新操作可能需要较长时间，请耐心等待
5. 确保有足够的网络带宽和稳定性
